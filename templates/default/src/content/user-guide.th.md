# คู่มือผู้ใช้

## ผู้ใช้จะเห็นอะไรในระบบเอกสาร

ระบบเอกสารมี 3 ส่วนหลัก:

- แถบซ้าย: สารบัญของหน้าเอกสารปัจจุบัน
- ส่วนกลาง: เนื้อหาบทความ
- แถบขวา: ตัวนำทางข้ามหน้าเอกสาร

รูปแบบนี้ช่วยให้ผู้ใช้อยู่กับบริบทเดิมได้แม้หน้าเอกสารยาว

## โฟลว์การอ่านสำหรับผู้ใช้

1. เปิดหน้าเอกสารจาก navigator ด้านขวา
2. ใช้ TOC ด้านซ้ายเพื่อกระโดดไปหัวข้อย่อย
3. เลื่อนอ่านตามธรรมชาติ โดย active heading จะอัปเดตอัตโนมัติ
4. ใช้ปุ่ม back/forward ของเบราว์เซอร์เพื่อย้อนหน้าได้

## พฤติกรรมของ URL

- URL หน้าจะใช้ slug เช่น `/introduction` หรือ `/api-reference`
- ลิงก์หัวข้อย่อยจะเป็น hash เช่น `#getting-started`
- แชร์ deep link ตรงไปยังหน้าและหัวข้อได้

## หมายเหตุด้านการใช้งาน

- โครง heading ออกแบบมาให้อ่านและนำทางง่าย
- TOC โฟกัส H2 และ H3 เพื่อความชัดเจน
- Frontmatter เป็น optional และมี fallback text ให้

## แนวทางเขียนเนื้อหาแนะนำ

- แบ่ง section สั้น และตั้ง heading ให้สื่อความหมาย
- 1 section ต่อ 1 ประเด็นหลัก
- ใส่ตัวอย่างใกล้คำอธิบายที่เกี่ยวข้อง
- ใช้ภาษาตรงไปตรงมา หลีกเลี่ยงศัพท์ภายในทีมที่ไม่จำเป็น

## สำหรับทีมที่ดูแลระบบนี้

ก่อน publish เอกสาร:

1. ตรวจว่าทุกหน้ามี title และ description ชัดเจน
2. ตรวจ heading IDs และ TOC links ว่าถูกต้อง
3. ตรวจ navigation links ระหว่างหน้าหลัก
4. smoke test บนมือถือและเดสก์ท็อป
